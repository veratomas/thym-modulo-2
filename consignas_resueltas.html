Ejercicio 9 — Ordenamiento Burbuja vs sort() con violín

Descripción

El algoritmo Bubble Sort (burbuja) compara pares adyacentes e intercambia elementos si están en orden incorrecto, repitiendo el proceso hasta ordenar el vector completo. Su complejidad es \(O(n^2)\), lo que lo hace ineficiente para vectores grandes.

El comando sort() nativo de R implementa un algoritmo híbrido altamente optimizado (basado en shellsort o radix sort), con complejidad promedio \(O(n \log n)\).

Implementación

library(microbenchmark)

# Muestra de 20.000 números aleatorios
set.seed(123)
x <- sample(1:100000, 20000)

# Función burbuja
burbuja <- function(x) {
  n <- length(x)
  for (j in 1:(n - 1)) {
    for (i in 1:(n - j)) {
      if (x[i] > x[i + 1]) {
        temp   <- x[i]
        x[i]   <- x[i + 1]
        x[i + 1] <- temp
      }
    }
  }
  return(x)
}

# Benchmark (pocas repeticiones para burbuja por su lentitud)
resultado <- microbenchmark(
  burbuja = burbuja(x),
  sort_R  = sort(x),
  times   = 3
)

print(resultado)
## Unit: milliseconds
##     expr         min           lq         mean       median           uq

##   sort_R     1.06746     1.104113     1.195648     1.140766     1.259742
##           max neval
##  31832.800438     3

Conclusión

El gráfico de violín evidencia que sort() es órdenes de magnitud más rápido que Bubble Sort para \(n = 20.000\). Mientras sort() opera en microsegundos, la burbuja demanda decenas de segundos, confirmando la diferencia teórica entre \(O(n^2)\) y \(O(n \log n)\).

K-Means — Introducción Teórica y Medición de Performance

Introducción teórica

El algoritmo K-Means es un método de aprendizaje no supervisado que particiona un conjunto de datos en \(k\) grupos (clusters) de forma iterativa:

  1. Se seleccionan \(k\) centroides al azar.
  2. Cada observación se asigna al centroide más cercano (distancia euclídea).
  3. Se recalcula cada centroide como el promedio de los puntos asignados a él.
  4. Se repiten los pasos 2–3 hasta convergencia (sin reasignaciones) o hasta alcanzar el máximo de iteraciones (10 por defecto en R).

El criterio que minimiza es la suma de cuadrados intra-cluster:

\[SS(k) = \sum_{i=1}^{n} \sum_{j=1}^{p} (x_{ij} - \bar{x}_{kj})^2\]

Consideraciones importantes:

  • Todas las variables deben ser continuas y conviene estandarizarlas antes de aplicar k-means (función scale()), ya que el algoritmo es sensible a la escala.
  • El número de clusters \(k\) es un hiperparámetro; se puede estimar con el método del codo (wss) o con el paquete NbClust.
  • Los resultados pueden variar entre ejecuciones; se recomienda usar set.seed() y el parámetro nstart (configuraciones iniciales aleatorias).
